Weekend reality check — demand vs supply

Question raised: only 6% of pipeline students requested "Weekend Only," so why does our heatmap show weekend activity? Here's the comparison — using only the trustworthy Availability (Readable - Local) field per the data-quality analysis (the Eastern-rendered column has midnight-wraparound bugs that would corrupt this).

Aligned with the cited read. Only 6.3% of pipeline (with availability filled) is Weekend Only — matches the ~6% figure cited. We're scheduling 15.4% of classes on weekends, which is BELOW the 55.5% who'd accept weekends (Weekend Only + Both/Mixed). So no over-staffing of weekends — we're slotting weekend-flexible students into weekend openings.

Data quality caveats: • 61% of pretrial responses haven't completed the Availability field yet (293 of 484) — they're excluded from the % calculation, so denominators stay consistent across teams. • We use only the Availability (Readable - Local) column. The Availability (Readable) Eastern column has 3 known bugs (midnight UTC wraparound, AM/PM at 12-hour boundary, inconsistent DST handling) and is not trusted here. • Keyword-scan is approximate — we look for "Sat/Sun/weekend" vs "Mon-Fri/weekday" mentions in the rendered text.

Method: For each student with Pretrial availability filled + an XP id + a resolved state, parse the Availability (Readable - Local) ranges into (day-of-week, time-of-day) windows. Excluded: 121 had availability but no scheduled future events, 5 couldn't be linked (missing deal/xp_id/state), 0 had unparseable availability text.
